# 2026-06-09: M2 Shipped — Orchestra MCP SDLC Server Live

## Overview

Today we bootstrapped the orchestra repo from scratch and shipped M2: a live, authenticated
Cloudflare Worker serving SDLC tools over MCP HTTP. Any project can now point `.mcp.json` at
`https://orchestra-mcp-sdlc.mpazbot.workers.dev/mcp` and get the full orchestra pipeline
immediately — no local setup required.

We ran the entire SDLC on itself: scaffold → roadmap → plan → implement → review → merge.
This devlog is written from inside that same pipeline.

## What Shipped

**`apps/mcp-sdlc`** — Stateless Cloudflare Worker. Three tools:

| Tool | What it does |
|------|-------------|
| `orchestra_get_prompt` | Returns the system prompt for any SDLC stage |
| `orchestra_get_gates` | Returns prerequisite stages before advancing |
| `orchestra_list_stages` | Returns the full pipeline in order |

**Auth:** `Authorization: Bearer <key>` on `/mcp`. `/health` stays open. Same key pattern as
savvy-agents — shared UUID stored as a Cloudflare secret, referenced via env var in `.mcp.json`.

**`packages/core`** — Shared stage prompts (`intake → prd → spec → gherkin → plan → execute`),
gate definitions, and artifact schemas. The Worker imports from here.

**`skills/`** — 17 SDLC skills moved from agentic-factory into the repo. The skills and the
infrastructure now live together.

## Key Decisions

**Stateless MCP, local artifacts.** The Worker serves prompts and logic. Claude Code writes
all `.orchestra/` files locally. No KV, no Durable Objects on the MCP. State lives in the repo.
This keeps the Worker cheap, fast, and easy to reason about.

**`--no-ff` merges preserve pipeline history.** Each work item (`impl/{ticket-id}`) merges as
a distinct unit. `git log --oneline` reads like a changelog.

**TDD ordering enforced by review.** The first review caught unit tests committed after the
source fix. Fixed via cherry-pick reorder before merge. The review skill's gate is real.

**Shared API key with savvy-agents.** One key across deployments. Simple for now — can scope
per-project later if needed.

## What the Pipeline Felt Like

Running orchestra on orchestra revealed two things:

1. **The gates work.** The review skill caught a real issue (integration test running in default
   suite without a guard) that would have silently broken CI. It was flagged, fixed, and the
   branch was cleaned up before merge. This is the point.

2. **The commit order check is meaningful.** Cherry-picking to reorder takes 2 minutes.
   Having a history that reads red-then-green is worth it — it's the difference between
   "tests exist" and "tests drove the implementation."
