# relnotes

> A command-line tool that turns a git repository's commit history between two
> tags into polished, human-readable release notes in markdown.

## Brief

**Vision:** A maintainer runs `relnotes v1.2.0 v1.3.0` and gets clean,
grouped, markdown release notes — features, fixes, and breaking changes
sorted out, noise commits dropped — ready to paste into a GitHub release or
CHANGELOG.

**Audience:** Open-source and internal maintainers who tag releases with git
and want changelog-quality notes without hand-curating every commit.
